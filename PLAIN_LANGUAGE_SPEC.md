# GEO-LAB Plain Language Spec v1

См. также: `TIMELINE_ARCHITECTURE.md` (контракт тайм-линий и scope-изоляции).

## Product mission
Сделать продукт, в котором ребёнок 10 лет за 30 минут понимает:
1. Что сейчас происходит в мире
2. Кто в этом участвует
3. Почему это важно
4. На каких фактах это основано

## Core rule
Интерфейс должен быть понятен **без специальных режимов и кнопок "упростить"**.

## UX principles
- Один экран = одна главная мысль
- Любой вывод рядом с подтверждением
- Любой термин объясняется простыми словами
- Никакого проф-жаргона в видимом тексте
- Сначала "что случилось", потом "почему важно", потом "чем подтверждается"

## Text style rules
- Короткие фразы: до 12 слов в предложении, где возможно
- Глаголы действия: "произошло", "влияет", "подтверждается"
- Избегать абстракций: вместо "эскалация" → "рост напряжения"
- Не использовать англоязычные UI-подписи

## Typography system (strict)
Только 3 размера текста:
- `t-display` — заголовок
- `t-body` — основной текст
- `t-meta` — служебная подпись

Целевая шкала: x1.8 между уровнями.

## Mandatory widget structure

### 1) Главная карточка сюжета
- Что случилось
- Почему важно
- Что смотреть дальше

### 2) Страница сюжета
- Простое описание
- Участники
- Хронология
- Подтверждения
- Возможные последствия

### 3) Карта связей
- Понятные имена связей
- Подпись "уверенность" и источник
- Режимы просмотра: Простой / Схема / Объём

### 4) Подтверждения
- Кто/что подтверждает
- Какая связь
- Насколько уверены
- Основание (источники)

## Forbidden words in visible UI (replace)
- case → сюжет
- narrative → сюжет
- evidence → подтверждение
- confidence (abbr/conf) → уверенность
- timeline → хронология
- graph → карта связей
- quality → качество данных

## Acceptance checklist for each screen
- [ ] Ребёнок понимает экран без помощи взрослого
- [ ] Нет непояснённых терминов
- [ ] Нет английских подписей
- [ ] Нет больше 3 размеров текста
- [ ] Любой вывод привязан к источнику
